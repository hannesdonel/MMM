Welcome! If you're here, you want to know how things work here. Lucky you, this documentation will help you with that.

A search query returning movies by genre or/and actor. /movies/?genre=[genreID] or/and /movies/?actor=[actor] GET The search query should contain "genreID" or/and "actor". A JSON object holding all movies assigned to a specific genre or/and actor.
Return data about a single movie by title. /movies/:itle GET Specify movie title in URL. The query is case sensitive! A JSON object holding data about a single movie by title.
Return data about all genres. /genres GET - A JSON object holding data about all genres:

{
"name": "Sci-Fi",
"description": "Lorem ipsum..."
}
Return data about a genre by name. /genre/:name GET Specify genre name in URL. The query is case sensitive! A JSON object holding data about a genre by name.
Return data about all directors. /directors GET - A JSON object holding data about all directors:

{
"name": "Al Capone",
"bio": "Al Capone was the most funniest movie director ever. Especially when it comes to Mafia movies.",
"birth_year": "1927",
"death_year": "2027"
}
Return data about a director by name. /directors/:name GET Specify director name in URL. The query is case sensitive! A JSON object holding data about a director.
Allow new users to register. /users POST A JSON object holding data containing first name, last name, age, mail adress and birthday. Example:

{
"user_name" : STRING,
"password" : STRING,
"email" : STRING,
"birth_date" : STRING (YYYY-MM-DD)
} 		A verification message if the registration has been succesfull or not. In case of success you'll get the new user as a JSON object.
Allow users to deregister by ID. /users/:id DELETE The ID of the user to be deleted should be put after /users/ in the URL." A verification message if the deregistration has been succesfull or not.
Allow users to see their information by name. /users/:user_name GET The name of the user to be shown should be put after /users/ in the URL. A JSON object holding all data about a specific user.
Allow users to update their user data by ID. /users/:id PUT A JSON object holding new user data. Example:

{
"user_name" : STRING,
"password" : STRING,
"email" : STRING,
"birth_date" : DATE
" } 		A verification message that a new username was succesfully assigned or failed.
Allow users to add a movie to their list of favorites. /users/:user_name/favorites/:movieID PUT user_name indicates which user's favorites to be updated. :movieID provides ID of the movie to be added. A message wether the film has been added succesfully to the favorites or not.
Allow users to remove a movie from their list of favorites. /users/:user_name/favorites/:movieID DELETE user_name indicates which user's favorites to be updated. :movieID provides ID of the movie to be removed. A message wether the film has been aremoved succesfully from the favorites or not.
